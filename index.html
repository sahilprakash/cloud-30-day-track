Thirty days to ship infrastructure you can rebuild from code.

Four themed weeks — networking, AWS, automation, and operations — that build on each other, ending in a capstone you deploy, monitor, and tear down entirely from scripts. Every day has a concrete lab. Check days off as you go; your progress is saved on this device.

Duration 30 days · ~2 hrs/day Level Intermediate Format Read → Lab → Checkpoint Ends in Automated capstone

How to use this

  1. Read the day's objectives, then do the hands-on lab — the lab is where the learning sticks.
  2. Answer the checkpoint out loud before moving on. If you can't, revisit.
  3. Tick the checkbox when the lab runs clean. Keep everything in one Git repo.
  4. Don't skip the weekend review/project days — they wire the week together.
